# 人格建模规范（V1）

## 文档目标

本规范用于定义《失忆宝宝》项目的人格建模体系。

人格建模模块负责从角色素材中提取人格特征，构建可计算、可解释、可更新的人格模型，为行为预测、角色对话、人格成长等模块提供基础数据。

---

# 一、设计原则

## 1. 人格不是标签

错误示例：

```json
{
  "personality": "3号人格"
}
```

正确示例：

```json
{
  "achievement_drive": 0.82,
  "curiosity": 0.74,
  "risk_preference": 0.46
}
```

人格首先是特征向量，其次才是人格分类。

---

## 2. 人格是概率

人格分析结果必须以概率形式表达。

示例：

```json
{
  "type3": 0.62,
  "type5": 0.21,
  "type6": 0.12
}
```

禁止输出绝对结论。

---

## 3. 所有人格结论必须可追溯

任何人格判断必须绑定证据。

示例：

```json
{
  "trait": "achievement_drive",
  "score": 0.82,
  "evidence": [
    "长期参加竞赛",
    "反复强调成功目标",
    "主动承担高难任务"
  ]
}
```

---

# 二、人格模型结构

人格模型共分为五层：

```text
素材
↓
人格特征层
↓
大五人格层
↓
九型人格层
↓
价值观层
↓
行为预测层
```

---

# 三、人格特征层（核心层）

V1采用10个核心人格特征。

所有人格分析均基于这10个维度展开。

---

## 1. 成就驱动

字段：

achievement_drive

定义：

追求成功、竞争和结果导向的程度。

高分表现：

* 喜欢竞争
* 注重结果
* 希望获得认可
* 目标感强

---

## 2. 求知欲

字段：

curiosity

定义：

探索未知和学习新知识的意愿。

高分表现：

* 喜欢研究问题
* 热爱学习
* 对新事物好奇
* 主动探索

---

## 3. 风险偏好

字段：

risk_preference

定义：

接受不确定性和冒险行为的倾向。

高分表现：

* 敢于尝试
* 喜欢创新
* 不惧失败

---

## 4. 安全需求

字段：

security_need

定义：

对稳定、秩序和保障的需求程度。

高分表现：

* 风险规避
* 喜欢规划
* 注重安全感

---

## 5. 控制欲

字段：

dominance

定义：

影响环境和主导局面的倾向。

高分表现：

* 喜欢领导
* 喜欢决策
* 讨厌失控

---

## 6. 同理心

字段：

empathy

定义：

理解和感受他人情绪的能力。

高分表现：

* 善于共情
* 乐于帮助他人
* 关注他人感受

---

## 7. 独立性

字段：

independence

定义：

自主思考和自主决策的倾向。

高分表现：

* 不依赖他人
* 坚持自我判断
* 独立解决问题

---

## 8. 责任感

字段：

responsibility

定义：

履行义务和坚持承诺的能力。

高分表现：

* 自律
* 守信用
* 做事认真

---

## 9. 创造力

字段：

creativity

定义：

产生新想法和突破传统思维的能力。

高分表现：

* 创新
* 发散思维
* 喜欢尝试不同方案

---

## 10. 社交需求

字段：

social_need

定义：

与他人互动和建立关系的需求程度。

高分表现：

* 喜欢社交
* 乐于表达
* 主动建立联系

---

# 四、大五人格映射层

人格特征层自动映射为大五人格。

## 开放性（O）

主要来源：

* curiosity
* creativity

---

## 责任心（C）

主要来源：

* responsibility
* achievement_drive

---

## 外向性（E）

主要来源：

* social_need
* dominance

---

## 宜人性（A）

主要来源：

* empathy

---

## 神经质（N）

主要来源：

* security_need

---

输出格式：

```json
{
  "openness": 78,
  "conscientiousness": 84,
  "extraversion": 62,
  "agreeableness": 55,
  "neuroticism": 37
}
```

---

# 五、九型人格映射层

基于人格特征推导九型人格概率。

示例：

3号成就者：

* achievement_drive ↑
* social_need ↑

5号观察者：

* curiosity ↑
* social_need ↓

6号忠诚者：

* security_need ↑

输出格式：

```json
{
  "type1": 0.03,
  "type2": 0.06,
  "type3": 0.61,
  "type4": 0.04,
  "type5": 0.18,
  "type6": 0.07,
  "type7": 0.00,
  "type8": 0.00,
  "type9": 0.01
}
```

---

# 六、价值观层

价值观独立于人格。

人格决定倾向。

价值观决定取舍。

V1包含5个价值观维度：

```json
{
  "career": 0.85,
  "family": 0.70,
  "freedom": 0.65,
  "wealth": 0.75,
  "status": 0.80
}
```

定义：

career：事业优先程度

family：家庭优先程度

freedom：自由优先程度

wealth：财富优先程度

status：社会地位优先程度

---

# 七、人格证据系统

所有人格分析结果必须保留证据链。

格式：

```json
{
  "trait": "achievement_drive",
  "score": 0.82,
  "confidence": 0.75,
  "evidence": [
    {
      "source": "视频采访",
      "content": "我希望成为行业第一"
    },
    {
      "source": "个人文章",
      "content": "竞争让我充满动力"
    }
  ]
}
```

---

# 八、人格成长机制

人格数据禁止直接覆盖。

采用人格时间轴机制。

示例：

```json
{
  "2022": {
    "risk_preference": 0.82
  },
  "2024": {
    "risk_preference": 0.63
  },
  "2026": {
    "risk_preference": 0.55
  }
}
```

系统能够观察人格变化趋势。

# V1核心结论

人格建模采用：

```text
素材
↓
人格特征向量
↓
大五人格
↓
九型人格
↓
价值观模型
↓
行为预测
```

其中：

人格特征向量 = 核心数据层

大五人格 = 性格结构层

九型人格 = 动机解释层

价值观模型 = 决策解释层

四者共同组成角色人格模型。
