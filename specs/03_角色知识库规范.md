# 角色知识库规范（V1）

## 文档目标

定义《失忆宝宝》角色知识库结构。

知识库负责存储：

* 角色知道什么
* 角色经历过什么
* 角色相信什么
* 角色与谁有关联

知识库是角色人格之外最重要的数据层。

---

# 一、设计原则

## 原则1：人格 ≠ 知识

人格决定：

```text
怎么想
```

知识决定：

```text
知道什么
```

例如：

两个5号人格角色。

一个是程序员。

一个是医生。

人格可能接近。

知识库完全不同。

---

## 原则2：知识必须结构化

禁止：

```text
把所有资料直接丢给LLM
```

应拆解成：

```text
人物
事件
关系
观点
时间线
```

---

## 原则3：所有知识必须可追溯

知识必须保留来源。

例如：

```json
{
  "fact": "毕业于清华大学",
  "source": "采访视频",
  "confidence": 0.92
}
```

---

# 二、知识库总体结构

```text
KnowledgeBase
├── Profile（基础信息）
├── Relationships（人物关系）
├── Events（重要事件）
├── Beliefs（观点体系）
├── Timeline（人生时间轴）
├── Facts（事实库）
└── Evidence（证据库）
```

---

# 三、基础信息（Profile）

## 作用

记录角色最基础的身份信息。

## 数据结构

```json
{
  "name": "",
  "alias": [],
  "gender": "",
  "birthday": "",
  "occupation": "",
  "nationality": "",
  "education": "",
  "location": "",
  "description": ""
}
```

## 示例

```json
{
  "name": "张三",
  "occupation": "创业者",
  "education": "计算机科学",
  "description": "连续创业者"
}
```

---

# 四、人物关系（Relationships）

## 作用

描述角色与其他人物之间的关系。

## 关系类型

```text
父母
兄弟姐妹
朋友
同事
伴侣
导师
竞争对手
合作伙伴
```

## 数据结构

```json
{
  "person_id": "",
  "name": "",
  "relationship": "",
  "closeness": 0.0,
  "description": ""
}
```

## 示例

```json
{
  "name": "李四",
  "relationship": "朋友",
  "closeness": 0.85,
  "description": "大学室友"
}
```

  "impact": 0.95

---

# 八、人生时间轴（Timeline）

## 作用

按照时间组织角色经历。

## 数据结构

```json
{
  "year": "",
  "events": []
}
```

## 示例

```json
{
  "2020": [
    "创业",
    "获得融资"
  ],
  "2021": [
    "公司扩张"
  ]
}
```

---

# 九、证据库（Evidence）

## 作用

保存所有知识来源。

保证知识可解释、可验证。

## 数据结构

```json
{
  "source_type": "",
  "source_name": "",
  "content": "",
  "confidence": 0.0
}
```

## source_type

```text
video
audio
article
wiki
social_media
chat
user_upload
```

## 示例

```json
{
  "source_type": "video",
  "source_name": "人物采访",
  "content": "我从大学开始创业",
  "confidence": 0.91
}
```

# 十一、知识更新机制

## 新知识出现

```text
新视频
新文章
新访谈
```

↓

```text
知识抽取
```

↓

```text
冲突检测
```

↓

```text
知识更新
```

---

## 更新原则

### 低置信度被高置信度覆盖

例如：

```json
{
  "occupation": "工程师",
  "confidence": 0.55
}
```

更新为：

```json
{
  "occupation": "创业者",
  "confidence": 0.92
}
```

---

### 保留历史版本

```json
{
  "version": [
    {
      "value": "工程师",
      "time": "2022"
    },
    {
      "value": "创业者",
      "time": "2024"
    }
  ]
}
```

# V1核心结论

知识库负责回答三个问题：

```text
角色知道什么？

角色经历过什么？

角色相信什么？
```

人格模块负责：

```text
角色会怎么想？
```

知识库负责：

```text
角色有什么信息可以用于思考？
```

两者共同构成角色认知系统。
