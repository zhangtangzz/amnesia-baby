# 行为预测规范（V1）

## 文档目标

定义《失忆宝宝》行为预测系统。

行为预测模块负责根据角色人格、价值观、知识库、历史记忆和当前情境，推理角色在未知场景下最可能采取的行为与回答。

---

# 一、设计目标

行为预测不是预测未来。

行为预测的本质是：

```text
给定一个情境

推理该角色最可能如何思考
最可能如何决策
最可能如何表达
```

即：

```text
Situation
↓
Reasoning
↓
Decision
↓
Response
```

---

# 二、核心原则

## 原则1：先决策，再回答

错误流程：

```text
用户提问
↓
LLM直接回答
```

正确流程：

```text
用户提问
↓
角色思考
↓
角色决策
↓
角色表达
↓
最终回答
```

---

## 原则2：行为必须符合人格

行为不能脱离人格模型。

例如：

角色：

```json
{
  "risk_preference": 0.2,
  "security_need": 0.9
}
```

面对创业问题：

系统不应直接鼓励冒险。

---

## 原则3：行为必须符合价值观

人格决定倾向。

价值观决定选择。

例如：

角色A：

```json
{
  "career": 0.9,
  "family": 0.3
}
```

角色B：

```json
{
  "career": 0.3,
  "family": 0.9
}
```

面对异地工作机会：

决策可能完全不同。

---

## 原则4：行为必须符合历史记忆

角色不能遗忘已经发生的事件。

例如：

用户：

```text
我们昨天刚聊过创业计划
```

角色必须能够引用相关记忆。

---

# 三、行为预测公式

V1版本采用：

```text
Behavior =
f(
Personality,
Values,
Knowledge,
Memory,
Situation
)
```

即：

```text
行为 =
f(
人格,
价值观,
知识库,
历史记忆,
当前情境
)
```

---

# 四、输入模型

## 1. Personality

来自人格建模模块。

示例：

```json
{
  "achievement_drive": 0.82,
  "curiosity": 0.74,
  "risk_preference": 0.35,
  "security_need": 0.81
}
```

---

## 2. Values

价值观模型。

示例：

```json
{
  "career": 0.88,
  "family": 0.65,
  "freedom": 0.52,
  "wealth": 0.75
}
```

---

## 3. Knowledge

角色知识库。

包含：

* 生平经历
* 人际关系
* 世界观设定
* 重要事件

---

## 4. Memory

长期记忆。

包含：

* 用户历史对话
* 用户共同经历
* 已产生的新记忆

---

## 5. Situation

当前场景。

示例：

```json
{
  "topic": "创业",
  "emotion": "焦虑",
  "context": "用户准备辞职"
}
```

---

# 五、行为预测流程

## Step1：情境理解

分析当前问题。

例如：

用户：

```text
我要不要辞职创业？
```

输出：

```json
{
  "type": "career_decision",
  "risk_level": "high",
  "importance": "high"
}
```

---

## Step2：人格影响分析

分析人格对行为的影响。

例如：

```json
{
  "risk_preference": 0.2,
  "security_need": 0.9
}
```

得到：

```text
倾向保守决策
倾向规避风险
```

---

## Step3：价值观影响分析

例如：

```json
{
  "career": 0.9,
  "family": 0.9
}
```

可能产生价值冲突：

```text
事业发展
VS
家庭稳定
```

---

## Step4：知识检索

从知识库检索相关内容。

例如：

```text
角色曾创业失败
```

或者：

```text
角色公开支持创业
```

---

## Step5：记忆检索

从长期记忆检索相关事件。

例如：

```text
用户此前提过资金不足
```

---

## Step6：生成决策

输出角色决策。

示例：

```json
{
  "decision": "暂缓创业",
  "confidence": 0.81,
  "reason": [
    "风险过高",
    "资金不足",
    "角色风险偏好较低"
  ]
}
```

---

## Step7：生成表达

决策不直接展示。

而是转换成角色语言风格。

例如：

决策：

```text
暂缓创业
```

表达：

```text
如果是我，我会再准备半年。

创业本身没问题，但现在的准备还不够充分。
```

# 七、一致性验证系统

## 目标

检查行为是否符合角色设定。

---

### 检查项1

人格一致性

例如：

```text
高风险厌恶人格
却频繁鼓励赌博
```

判定异常。

---

### 检查项2

价值观一致性

例如：

```text
家庭优先
却持续忽视家庭
```

判定异常。

---

### 检查项3

知识一致性

例如：

```text
角色明明认识某人
却说不认识
```

判定异常。

---

### 检查项4

记忆一致性

例如：

```text
昨天刚说过的话
今天完全遗忘
```

判定异常。

---

# 八、行为成长机制

角色行为不是固定的。

系统需要记录行为历史。

例如：

```json
{
  "event": "创业",
  "decision_2024": "拒绝",
  "decision_2026": "支持"
}
```

系统分析：

```text
人格变化？

价值观变化？

知识变化？

环境变化？
```

形成行为成长轨迹。

```text

      ↓

(人格分析)

      ↓

      ↓

      ↓

      ↓

(一致性验证)

      ↓

Final Response
```

# 十、V1核心结论

行为预测不是回答生成。

行为预测是：

```text
情境
↓
人格影响
↓
价值观影响
↓
知识影响
↓
记忆影响
↓
决策生成
↓
语言表达
```

其中：

Decision Agent

是整个系统最核心的Agent。

因为用户真正想体验的不是：

“角色会说什么”

而是：

“角色会怎么想、怎么选、怎么做”。
