# 记忆系统规范（V1）

## 文档目标

定义《失忆宝宝》记忆系统结构。

记忆系统负责存储：

* 用户与角色之间发生过什么
* 角色记住了什么
* 用户告诉过角色什么
* 角色与用户共同创造了什么经历

记忆系统是角色长期陪伴能力的核心。

---

# 一、设计原则

## 原则1：知识不等于记忆

知识：

```text
角色本来就知道的事情
```

例如：

```text
出生地
职业经历
创业历史
人物关系
```

---

记忆：

```text
角色与用户互动产生的事情
```

例如：

```text
用户昨天面试失败
用户喜欢喝咖啡
用户计划创业
```

---

## 原则2：记忆具有时间属性

必须知道：

```text
什么时候发生
```

例如：

```text
昨天

上周

三个月前
```

---

## 原则3：记忆具有重要程度

不是所有内容都永久保存。

例如：

```text
今天吃了什么
```

价值较低。

---

而：

```text
第一次创业失败
```

价值较高。

---

# 二、记忆系统结构

```text
Memory System

├── Short Memory
├── Long Memory
├── Character Memory
├── Shared Memory
├── Emotional Memory
└── Memory Timeline
```

---

# 三、短期记忆（Short Memory）

## 作用

保存当前会话上下文。

类似：

```text
最近20~50轮对话
```

---

## 示例

用户：

```text
我准备创业
```

5分钟后：

```text
需要准备多少钱？
```

角色需要知道：

```text
当前讨论的是创业
```

---

## 数据结构

```json
{
  "session_id": "",
  "messages": [],
  "expire_time": ""
}
```

---

# 四、长期记忆（Long Memory）

## 作用

保存用户长期信息。

---

## 示例

用户：

```text
我是Java开发者
```

一周后：

```text
最近在学习LangGraph
```

角色应该能够关联：

```text
你之前提过自己是Java开发者
```

---

## 数据结构

```json
{
  "memory_id": "",
  "content": "",
  "importance": 0.0,
  "created_at": ""
}
```

---

# 五、角色记忆（Character Memory）

## 作用

记录角色自身经历。

---

## 来源

```text
角色原始资料

新增资料

人格成长记录
```

---

## 示例

```text
角色第一次创业

角色参加比赛

角色获得融资
```

---

## 数据结构

```json
{
  "event_id": "",
  "description": "",
  "time": "",
  "impact": 0.0
}
```

---

# 六、共享记忆（Shared Memory）

## 作用

记录角色与用户共同经历。

---

## 示例

用户：

```text
今天面试失败了
```

角色：

```text
安慰用户
```

几天后：

```text
面试结果怎么样？
```

角色应该能够提起：

```text
上次面试的事情
```

---

## 数据结构

```json
{
  "memory_id": "",
  "participants": [
    "user",
    "character"
  ],
  "content": "",
  "created_at": ""
}
```

---

# 七、情绪记忆（Emotional Memory）

## 作用

记录重要情绪事件。

---

## 示例

用户：

```text
第一次失恋
```

角色：

```text
记录情绪状态
```

未来：

```text
再次提及感情话题
```

角色能够理解背景。

---

## 数据结构

```json
{
  "emotion": "",
  "intensity": 0.0,
  "event": "",
  "time": ""
}
```

---

# 八、记忆时间轴（Memory Timeline）

## 作用

---

## 示例

```text

讨论创业

2026-03

模拟面试
```

---

## 数据结构


# 九、记忆重要度机制

## Importance Score

范围：

```text
0~1
```

---

### 低重要度

```text
今天吃什么

天气如何

闲聊内容
```

分数：

```text
0.1~0.3
```

---

### 中重要度

```text
学习计划

职业规划

兴趣爱好
```

分数：

```text
0.4~0.7
```

---

### 高重要度

```text
创业

重大决定

情感事件

人生转折
```

分数：

```text
0.8~1.0
```

# 十二、遗忘机制

## 原则

不是所有记忆永久保存。

---

低重要度记忆：

```text
自动衰减
```

---

高重要度记忆：

```text
永久保留
```

---

公式：

```text
Memory Score
=
Importance × Recency × Frequency
```

其中：

```text
Importance
重要程度

Recency
最近访问时间

Frequency
被引用次数
```

# V1核心结论

知识库回答：

```text
角色知道什么？
```

人格模型回答：

```text
角色会怎么想？
```

记忆系统回答：

```text
角色记得什么？
```

三者共同构成角色认知系统。

行为预测实际上依赖：

```text
人格
+
知识
+
记忆
+
情境
```

而记忆系统负责让角色拥有连续的人生体验，而不仅仅是一轮对话。
