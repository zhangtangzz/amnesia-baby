# 一致性验证规范（V1）

## 文档目标

定义《失忆宝宝》角色一致性验证系统。

一致性验证模块负责检查角色输出是否符合：

* 人格设定
* 价值观设定
* 知识库设定
* 记忆设定
* 历史行为

保证角色长期对话过程中保持稳定的人格与认知。

---

# 一、为什么需要一致性验证

## 问题

传统AI角色经常出现：

```text
今天支持创业

明天反对创业

后天又支持创业
```

或者：

```text
昨天说自己创业过

今天说自己从未创业
```

或者：

```text
前面表现得极其内向

后面突然变成社交达人
```

这些都会导致：

```text
角色崩坏
```

---

# 二、验证目标

一致性验证不负责生成答案。

负责判断：

```text
这个答案像不像这个角色
```

---

# 三、一致性验证结构

```text
Consistency Check

├── Personality Check
├── Value Check
├── Knowledge Check
├── Memory Check
├── Behavior Check
└── Confidence Check
```

---

# 四、人格一致性验证

## 目标

检查行为是否符合人格模型。

---

### 示例1

人格：

```json
{
  "risk_preference": 0.15,
  "security_need": 0.92
}
```

输出：

```text
辞职创业吧
不要考虑风险
```

判定：

```text
人格冲突
```

---

### 示例2

人格：

```json
{
  "social_need": 0.85
}
```

输出：

```text
我最讨厌社交
```

判定：

```text
人格冲突
```

---

## 输出格式

```json
{
  "score": 0.82,
  "conflict": false
}
```

---

# 五、价值观一致性验证

## 目标

检查角色是否违背核心价值观。

---

### 示例

价值观：

```json
{
  "family": 0.95,
  "career": 0.40
}
```

行为：

```text
为了工作彻底放弃家庭
```

判定：

```text
价值观冲突
```

---

## 输出格式

```json
{
  "score": 0.75,
  "conflict": true,
  "reason": "违反家庭优先原则"
}
```

---

# 六、知识一致性验证

## 目标

检查回答是否符合知识库。

---

### 示例

知识库：

```text
角色毕业于北京大学
```

回答：

```text
我没有上过大学
```

判定：

```text
知识冲突
```

---

### 示例

知识库：

```text
角色创立过公司
```

回答：

```text
我从未创业
```

判定：

```text
知识冲突
```

---

# 七、记忆一致性验证

## 目标

检查是否遗忘重要记忆。

---

### 示例

昨天：

```text
用户面试失败
```

今天：

```text
用户再次提到面试
```

角色：

```text
什么面试？
```

判定：

```text
记忆缺失
```

---

### 示例

共享记忆：

```text
用户计划创业
```

角色：

```text
你之前提过创业计划
```

判定：

```text
记忆一致
```

---

# 八、行为一致性验证

## 目标

检查历史行为是否连续。

---

### 示例

历史行为：

```text
长期反对高风险投资
```

当前行为：

```text
建议卖房炒币
```

判定：

```text
行为异常
```

---

## 行为轨迹

记录：

```json
{
  "topic": "投资",
  "history": [
    "保守",
    "保守",
    "保守"
  ]
}
```

---

# 九、置信度验证

## 目标

检查当前回答是否有足够依据。

---

### 来源

```text
人格依据

知识依据

记忆依据

价值观依据
```

---

### 示例

```json
{
  "personality_support": 0.8,
  "knowledge_support": 0.7,
  "memory_support": 0.6,
  "final_confidence": 0.72
}
```

---

# 十、一致性评分

## 计算公式

```text
Consistency Score =
人格一致性
+
价值观一致性
+
知识一致性
+
记忆一致性
+
行为一致性
```

---

## 评分区间

### 0.9 ~ 1.0

```text
高度一致
```

---

### 0.7 ~ 0.9

```text
基本一致
```

---

### 0.5 ~ 0.7

```text
存在偏差
```

---

### 0.5以下

```text
角色崩坏
```

---

# 十一、修正机制

当检测到冲突：

```text
Consistency Agent
↓
发现冲突
↓
定位来源
↓
重新生成
```

---

示例：

```text
回答
↓
人格冲突
↓
重新推理
↓
生成新回答
```

---

# 十二、Agent架构

```text
Dialogue Agent
        ↓

Draft Response
（初始回答）

        ↓

Consistency Agent
（一致性验证）

        ↓

Personality Check
Value Check
Knowledge Check
Memory Check
Behavior Check

        ↓

Consistency Score

        ↓

Pass / Retry

        ↓

Final Response
```

# V1核心结论

一致性验证系统负责回答：

```text
这个回答像不像这个角色？
```

人格模型负责：

```text
角色会怎么想？
```

知识库负责：

```text
角色知道什么？
```

记忆系统负责：

```text
角色记得什么？
```

一致性系统负责：

```text
角色有没有说错话？
有没有人格崩坏？
有没有遗忘重要信息？
```

它是整个数字人格系统的质量控制中心（Quality Control Center）。
